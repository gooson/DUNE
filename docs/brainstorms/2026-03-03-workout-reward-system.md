# Brainstorm: 운동 보상 시스템 (PR 체계화 + 구간 달성 알림)

## Problem Statement

현재 운동 기록은 퍼스널 레코드(PR)와 달성 경험이 운동별로 일관되게 구조화되어 있지 않아, 사용자가 성취를 누적해서 체감하기 어렵다.
이번 기능의 핵심 목표는 다음 3가지다.

- 운동 지속률 증가
- 앱 재방문 증가
- 기록 갱신(PR 갱신) 유도

## Target Users

- 전 사용자 공통
- 특정 운동군(초보/중급/고급) 구분 없이, 기록 기반 동기부여가 필요한 모든 사용자

## Success Criteria

- 1차 핵심 지표: 구간 달성 알림 클릭률(CTR)
- CTR 개선이 확인되면 PR 확인 화면 진입률/재방문 지표를 2차로 확장 검토

## Proposed Approach

### 1) 기존 PR 체계 정규화

- 현재 PR 계산/표시 로직을 운동 타입 공통 모델로 정리한다.
- 운동별 핵심 PR 지표를 명시한다.
- 거리 기반: 최고 거리, 최고 속도/최저 페이스 등
- 걸음 기반: 최고 걸음 수
- 시간 기반: 최장 운동 시간
- 기타 운동: 운동별 대표 지표를 1개 이상 고정

### 2) 운동별 고정 구간(마일스톤) 테이블 도입

- 모든 운동 타입에 대해 고정 구간 테이블을 둔다.
- 예시
- 러닝: 5km, 10km, 15km, 20km
- 걸음: 10,000 / 15,000 / 20,000 보
- 구간은 개인화하지 않고 고정값으로 운영한다.
- 구간 달성 판단은 최종 확정 데이터만 기준으로 한다.

### 3) 구간 달성 알림 정책

- 운동 종료 후(세션 완료 시점) 달성 평가를 수행한다.
- 한 번의 운동에서 여러 구간을 동시에 넘겨도 대표 알림 1개만 발송한다.
- 대표 선택 규칙: 해당 세션에서 가장 높은 구간 또는 가장 의미 있는 PR 갱신 1건

### 4) 배지/레벨 보상 시스템 결합 (MVP 필수)

- 구간 달성 시 배지 획득 및 레벨 경험치(또는 점수) 누적을 제공한다.
- PR 갱신은 일반 구간 달성보다 높은 보상 가중치를 적용한다.
- 달성 이벤트(구간/PR/배지/레벨업)는 단일 히스토리로 기록한다.

### 5) 노출 화면 구성

- PR 현황 화면
- 운동별 현재 최고 기록, 다음 구간까지 남은 진행 상태
- 달성 히스토리 화면
- 구간 달성, PR 갱신, 배지 획득, 레벨업 타임라인

## Constraints

- 데이터 소스는 최종 확정 데이터만 허용한다(실시간 잠정값 제외).
- 모든 운동 타입 지원이 필요해 운동 분류/지표 정의 난이도가 높다.
- 알림은 대표 1개만 발송해야 하므로 다중 달성 요약 규칙이 필요하다.
- 운동 종료 직후 알림을 위해 동기화 지연(Watch/HealthKit 반영 지연) 처리 정책이 필요하다.

## Edge Cases

- 하나의 세션에서 다수 구간을 동시에 달성한 경우(대표 1개 선택)
- 동일 세션에서 구간 달성과 PR 갱신이 동시에 발생한 경우(우선순위 규칙)
- 중복 동기화로 동일 달성 이벤트가 2회 기록되는 경우(idempotency 필요)
- 운동 수정/삭제 시 이미 발송된 달성 알림 및 히스토리 정합성 문제
- 특정 운동의 대표 지표가 불명확하거나 데이터가 누락된 경우
- 알림 권한 미동의 시 인앱 히스토리만으로 보상 피드백을 유지해야 하는 경우

## Scope

### MVP (Must-have)

- [ ] 기존 PR 체계를 운동 공통 모델로 정리
- [ ] 모든 운동 타입에 대한 고정 구간 테이블 정의
- [ ] 운동 종료 후 구간 달성 평가 + 대표 알림 1개 발송
- [ ] 최종 확정 데이터 기준 판정(잠정 데이터 제외)
- [ ] PR 현황 화면 제공
- [ ] 달성 히스토리 화면 제공
- [ ] 배지/레벨 시스템 포함(달성 이벤트와 연동)

### Nice-to-have (Future)

- [ ] 사용자 난이도 기반 구간 추천(고정 구간 유지 + 추천만 제공)
- [ ] 리텐션 실험(A/B 테스트) 기반 알림 문구 최적화
- [ ] 주간/월간 챌린지와 배지/레벨 연동

## Resolved Questions

- 목적: 지속률 증가, 재방문 증가, 기록 갱신 유도
- 성공 지표: 알림 클릭률
- 대상: 전 사용자 공통
- 범위: 모든 운동
- 구간 방식: 고정값
- 다중 달성 알림: 대표 1개만 발송
- 알림 시점: 운동 종료 후
- 데이터 기준: 최종 확정 데이터만 사용
- 핵심 화면: PR 현황, 달성 히스토리
- 보상 요소: 배지/레벨 필수

## Open Questions

- 대표 알림 선택 우선순위(최고 구간 vs PR 갱신 우선)
- 배지/레벨 수식(경험치 곡선, 레벨 캡, 중복 달성 보상 정책)
