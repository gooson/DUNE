## 1. 섹션 통합/분리 재설계

### 문제 진단

현재 구조의 핵심 문제:

1. **코칭 메시지 파편화**: Morning Briefing, Coaching Card, Insight Cards가 모두 "오늘의 가이드" 성격인데 3곳에 흩어져 있음
2. **환경 정보 분산**: 날씨 카드가 독립 섹션인데, 모닝 브리핑에도 날씨 섹션이 있음 (중복)
3. **수면 부채 고립**: Sleep Deficit Badge가 하단에 혼자 떠 있어 맥락 없이 등장
4. **메트릭 과다**: Condition/Activity/Body 카드 + Pinned Metrics = 4개 메트릭 그리드가 연달아 나옴
5. **스크롤 피로**: 13개 섹션을 순차 스크롤 → 하단 메트릭에 도달하기 어려움

### 제안: 3-Zone 구조

**Zone A: 한눈에 (Glanceable)** — 스크롤 없이 보이는 영역

| 컴포넌트 | 변경 |
|---------|------|
| Condition Hero | 유지 (현행) |
| **Today's Brief** (NEW) | Morning Briefing + Coaching + Weather 통합 |

**Zone B: 오늘의 맥락 (Today's Context)** — 1스크롤 내

| 컴포넌트 | 변경 |
|---------|------|
| **Recovery & Sleep** (NEW) | Sleep Deficit + 수면 관련 인사이트 통합 |
| **Smart Insights** (NEW) | Insight Cards + Template Nudge 통합 |
| Health Q&A | 유지 (위치 조정) |

**Zone C: 내 데이터 (My Data)** — 깊이 탐색

| 컴포넌트 | 변경 |
|---------|------|
| Pinned Metrics | 유지 |
| Condition / Activity / Body | 유지 (핀에 안 된 것만) |

### 상세 설계

#### A-1. Today's Brief (통합 카드)

**통합 대상**: Morning Briefing Entry + Weather Card + Coaching Card

**근거**: 셋 다 "오늘 하루를 어떻게 보낼까"에 대한 답. 사용자는 아침에 한 곳에서 모든 맥락을 원함.

```
┌─────────────────────────────────────┐
│ ☀️ 맑음 24° · 체감 26°    용산구   │  ← 날씨 한 줄 요약
│─────────────────────────────────────│
│ 💪 오늘은 가볍게 움직이세요          │  ← 포커스 코칭 (1줄 핵심)
│                                     │
│ HRV가 평소보다 15% 낮고, 어제 하체   │  ← 근거 (2줄 이내)
│ 운동 후 회복 중입니다.               │
│─────────────────────────────────────│
│ 🌬️ 높은 습도 78% · PM₂.₅ 보통      │  ← 환경 알림 (조건부)
│ 실외 운동 시 수분 섭취에 신경 쓰세요  │
│─────────────────────────────────────│
│              자세히 보기 ›            │  ← 모닝 브리핑 시트 열기
└─────────────────────────────────────┘
```

**설계 원칙**:
- 날씨 + 코칭이 서로 맥락을 보강 (습도 높음 → 실외 주의)
- 환경 알림 줄은 조건부: 습도 >70%, 미세먼지 나쁨, UV 높음, 기온 극단 시만 표시
- "자세히 보기"로 기존 MorningBriefingView 시트 진입 (모든 세부 정보 보존)
- 날씨 상세는 시트 내 또는 별도 네비게이션 유지

**대안 A**: 카드 안에 날씨/코칭/환경을 세로 스택 (위 그림)
**대안 B**: 가로 스와이프 페이지 (날씨 | 코칭 | 주간 맥락) — Fitbit Morning Brief 스타일
**대안 C**: 접이식 카드 (기본: 1줄 요약, 탭하면 펼침)

#### A-2. Recovery & Sleep (통합 카드)

**통합 대상**: Sleep Deficit Badge + 수면 관련 Insight Cards

**근거**: 수면 부채가 고립되어 있으면 "그래서 뭘 해야 하지?"가 빠짐. 수면 인사이트와 합치면 문제+해법이 한 곳에.

```
┌─────────────────────────────────────┐
│ 🛏️ 수면 회복                        │
│─────────────────────────────────────│
│ ○ 수면 부채 2h 30m          보통    │  ← 게이지 + 레벨
│                                     │
│ 💡 어제 취침이 평소보다 1시간 늦었    │  ← 수면 인사이트
│    습니다. 오늘은 11시 전에 누워보세요│
│─────────────────────────────────────│
│           수면 상세 ›                │
└─────────────────────────────────────┘
```

**조건부 표시**: 수면 부채 > 0 또는 수면 관련 인사이트가 있을 때만

#### A-3. Smart Insights (통합 섹션)

**통합 대상**: Insight Cards + Template Nudge

**변경**: 둘 다 "앱이 발견한 것"을 알려주는 성격. 하나의 "Insights" 섹션 아래 카드 스택으로 통합.

```
┌─ 💡 인사이트 ───────────────────────┐
│                                     │
│ [인사이트 카드 1]              ✕    │
│ [인사이트 카드 2]              ✕    │
│ [템플릿 제안 카드]             ✕    │
│                                     │
└─────────────────────────────────────┘
```

- 인사이트가 없으면 섹션 자체 숨김
- 최대 3개 표시, 나머지는 "N개 더 보기"
- 각 카드 dismiss 가능 (현행 유지)

## 2. Today 탭 추가 아이디어

### 2-A. Adaptive Hero Message (적응형 히어로)

현재: 컨디션 점수 + 고정 구조의 링/스파크라인
제안: **시간대와 상태에 따라 히어로가 변신**

| 시간대 | 상태 | 히어로 메시지 |
|--------|------|-------------|
| 06-10시 | 컨디션 좋음 | "좋은 아침! 오늘 운동하기 딱 좋은 컨디션이에요" |
| 06-10시 | 컨디션 나쁨 | "오늘은 가볍게 쉬세요. 회복이 우선입니다" |
| 12-14시 | 운동 완료 | "오전 운동 완료! 스트레칭으로 마무리하세요" |
| 18-22시 | 수면 부채 있음 | "오늘은 일찍 자는 게 최고의 투자예요" |
| 22시+ | 아직 깨어있음 | "취침 시간이 다가오고 있어요" |

**구현**: 기존 `ConditionHeroView`의 narrative message를 시간대 + 오늘 활동 이력 기반으로 동적 생성. CoachingEngine 확장.

### 2-B. Quick Actions Row (빠른 동작)

현재: 대시보드에서 운동/기록 시작하려면 탭 전환 필요
제안: Hero 바로 아래 가로 스크롤 Quick Actions

```
[ 🏋️ 운동 시작 ] [ 🛏️ 수면 기록 ] [ ⚖️ 체중 기록 ] [ 📸 자세 촬영 ]
```

- 가장 자주 쓰는 액션 4개 고정
- 탭하면 바로 해당 시트/뷰로 이동
- 사용 빈도에 따라 순서 자동 정렬 (QuickStartPopularityService 활용)

### 2-C. Daily Progress Ring (하루 진행 상황)

현재: 주간 목표 진행률이 Hero 안에 작게 있음
제안: **오늘 하루의 3대 목표를 미니 링으로 시각화**

```
┌──────────────────────────────┐
│  🏃 활동     🛏️ 수면     💧 습관  │
│   ○ 65%      ○ 85%      ○ 3/5   │
│  4,200보    7h 12m     3개 완료  │
└──────────────────────────────┘
```

- 활동: 오늘 걸음 수 / 목표 (또는 운동 칼로리)
- 수면: 어젯밤 수면 / 목표
- 습관: Life 탭 오늘 완료율
- Apple Watch Activity Rings 와 유사한 직관적 UX
- Hero 카드 하단 또는 바로 아래 배치

### 2-D. Contextual Weather Alert

현재: 날씨 카드가 항상 같은 크기로 표시
제안: **주의가 필요한 날만 눈에 띄게, 평범한 날은 작게**

| 조건 | 표시 |
|------|------|
| 정상 날씨 | Today's Brief 내 1줄 요약 |
| 미세먼지 나쁨 | 주황색 배너 "실외 운동 주의" |
| 폭염/한파 | 빨간색 배너 + 수분/보온 팁 |
| 비/눈 | 파란색 배너 "실내 운동 추천" |
| 습도 >80% | 노란색 배너 "탈수 주의" |

평범한 날에 날씨 카드가 큰 공간을 차지하는 것을 방지.

### 2-E. Yesterday Recap Card (어제 한 줄 요약)

**새 카드**: 오전 시간대(6-12시)에만 표시

```
┌─ 어제 요약 ─────────────────────────┐
│ 🏋️ 상체 운동 45분 · 🛏️ 7h 수면     │
│ 컨디션 72 → 오늘 81 (+9)           │
└─────────────────────────────────────┘
```

- 어제 운동, 수면, 컨디션 변화를 한 줄로
- Oura/Whoop의 "yesterday in review" 패턴
- 오후에는 자동 숨김

### 2-F. Coaching Conversation Thread

현재: Health Q&A가 별도 시트로 분리
제안: **Today's Brief 카드 내에 "AI에게 물어보기" 인라인 진입점**

```
┌─ Today's Brief ─────────────────────┐
│ ...코칭 메시지...                    │
│                                     │
│ 💬 "왜 오늘 컨디션이 낮은가요?"      │  ← 탭하면 Q&A 시트
│    AI에게 물어보기 ›                 │
└─────────────────────────────────────┘
```

- 코칭 메시지의 맥락에서 자연스럽게 대화로 전환
- Health Q&A 카드를 별도로 두지 않아도 됨
- 코칭 메시지 기반 추천 질문 자동 생성

## 3. 활동 / 신체 탭 트렌디 아이디어

### 3-A. Activity 탭

#### (1) Training Load Ratio (급성/만성 비율)

Whoop/Garmin의 핵심 지표. 현재 TrainingVolume에 데이터가 있지만 비율 시각화가 약함.

```
┌─ 트레이닝 로드 ─────────────────────┐
│         이번 주 / 4주 평균           │
│                                     │
│    ████████████░░░  0.85            │
│    ◄ 감소 ·  적정  · 과부하 ►       │
│                                     │
│ 급성(7일): 2,400 AU                 │
│ 만성(28일): 2,800 AU                │
└─────────────────────────────────────┘
```

- 0.8-1.3이 "sweet spot" (과학적 근거: Gabbett TJ)
- 실시간 게이지 + 주간 추이 차트
- Training Readiness와 연동: 로드 비율이 과부하이면 readiness 경고

#### (2) Body Part Recovery Timeline

현재 Muscle Map은 정적 스냅샷. 제안: **시간 축 슬라이더**

```
┌─ 근육 회복 타임라인 ────────────────┐
│     [오늘]────[내일]────[모레]       │  ← 슬라이더
│                                     │
│     🟢 가슴: 완전 회복              │
│     🟡 하체: 내일 오후 회복 예상     │
│     🔴 등: 모레 회복 예상            │
│                                     │
│     ✅ 내일 추천: 가슴 + 어깨        │
└─────────────────────────────────────┘
```

- 각 근육 그룹별 예상 회복 시점
- 미래 날짜 슬라이드 → 해당 시점 회복 상태 시뮬레이션
- 운동 추천과 직접 연결

#### (3) Social Streak / Challenge

Gentler Streak + Strava 스타일. 현재 Consistency가 있지만 소셜 요소 없음.

- 친구와 주간 걸음 수 챌린지
- 운동 스트릭 공유 (SharePlay 확장)
- 리더보드 (선택적, Privacy-first)
- visionOS Spatial Training에서 함께 운동

#### (4) Exercise Intelligence Card

현재: 운동 추천이 단순 목록
제안: **"왜 이 운동을 추천하는지" 투명하게 설명**

```
┌─ 오늘의 추천 ───────────────────────┐
│ 🎯 상체 당기기 · 중강도 40분         │
│                                     │
│ 📊 근거:                            │
│ · 하체 피로도 높음 (어제 스쿼트)     │
│ · 등/이두 3일간 미사용              │
│ · 컨디션 82 → 중강도 적정           │
│ · 수면 7.5h → 회복 충분             │
│                                     │
│ [ 이 운동 시작 ]  [ 다른 추천 ]     │
└─────────────────────────────────────┘
```

- 추천 이유를 투명하게 (Explainable AI 트렌드)
- "다른 추천" → 대안 운동 + 이유 표시
- 사용자 신뢰도 향상

#### (5) Workout Consistency Heatmap

GitHub contribution graph 스타일:

```
       월  화  수  목  금  토  일
  W-4  ██  ░░  ██  ░░  ██  ██  ░░
  W-3  ██  ░░  ░░  ██  ██  ░░  ░░
  W-2  ██  ██  ░░  ██  ░░  ██  ░░
  W-1  ██  ░░  ██  ██  ░░  ██  ██
  Now  ██  ░░  ??  ??  ??  ??  ??
```

- 색상 강도 = 운동 볼륨/강도
- 패턴 인식: "화/목/토 루틴" 시각적 확인
- 현재 ConsistencyCard의 스트릭 숫자보다 풍부한 정보

### 3-B. 신체 (Wellness) 탭

#### (1) Body Composition Trend Visualization

현재: VitalCard 그리드로 Weight, BMI, Body Fat, LBM 개별 표시
제안: **통합 바디 컴포지션 차트**

```
┌─ 체성분 변화 (12주) ────────────────┐
│                                     │
│  체중 ━━━━━━━━━━━━━━━━━━━ 75.2kg   │
│  근육 ━━━━━━━━━━━━━━━━━━━ 62.1kg   │
│  체지방 ━━━━━━━━━━━━━━━━━ 13.1kg   │
│                                     │
│  △ 근육 +1.2kg  ▽ 체지방 -0.8kg    │
│  → 리컴포지션 진행 중 🎯            │
└─────────────────────────────────────┘
```

- 3개 라인을 한 차트에 (상대적 변화 강조)
- "리컴포지션" / "벌크" / "컷" 자동 판정
- 목표 설정 시 프로젝션 라인 표시

#### (2) Vitals Baseline Bands

현재: 각 VitalCard에 baseline delta만 표시
제안: **개인 정상 범위 밴드를 차트에 시각화**

```
┌─ 심박 변이도 (HRV) ────────────────┐
│                                     │
│  ┃ ░░░░░░░░░░░░░░░░░░░░░░░░░  80  │  ← 높은 정상
│  ┃ ████████████████████──●     62  │  ← 현재 값
│  ┃ ░░░░░░░░░░░░░░░░░░░░░░░░░  45  │  ← 낮은 정상
│  ┃─────────────────────────────     │
│    7일전                    오늘     │
│                                     │
│  📈 3일 연속 상승 추세              │
└─────────────────────────────────────┘
```

- 개인 baseline ± 1SD를 회색 밴드로
- 밴드 바깥 = 주의 필요 (빨간 점)
- Oura의 "Your Normal Range" 패턴

#### (3) Recovery Score (Wellness 전용)

현재: Wellness Score = Sleep + Condition + Body의 복합 점수
제안: **Recovery Focus 모드** 추가

- 운동 직후 → "회복 타이머" 활성화
- HRV 실시간 추적 (Apple Watch) → 회복 도달 시 알림
- 운동 후 24시간 동안 Recovery 카드가 Wellness 상단에 표시:

```
┌─ 🔄 회복 중 ────────────────────────┐
│ 어제 하체 운동 후 18시간 경과        │
│                                     │
│  HRV: 45 → 52 (▲ 회복 중)          │
│  RHR: 62 → 58 (▼ 안정화 중)        │
│                                     │
│  예상 완전 회복: 오늘 오후 3시       │
│  ████████████████░░░░  78%          │
└─────────────────────────────────────┘
```

#### (4) Wearable Data Freshness Dashboard

현재: 각 VitalCard에 freshness label이 있지만 분산됨
제안: **데이터 신선도 한눈에 보기** (설정 또는 Wellness 상단)

```
┌─ 데이터 상태 ───────────────────────┐
│ ✅ HR, HRV, RHR, Steps  (방금)      │
│ ✅ Sleep                 (오늘 아침) │
│ ⚠️ Weight               (3일 전)    │
│ ⚠️ Body Fat             (2주 전)    │
│ ❌ SpO2                 (미연결)     │
└─────────────────────────────────────┘
```

- stale 데이터에 대해 "기록하기" 바로가기
- 누락 데이터 자동 감지 → 기록 넛지

#### (5) Symptom + Vitals Correlation

현재: 부상 기록은 있지만 바이탈과의 상관관계 분석 없음
제안: **증상 기록 + 자동 상관 분석**

- "오늘 기분이 어때요?" 간단 체크인 (에너지, 스트레스, 통증)
- 주관적 느낌 vs 객관적 데이터 (HRV, 수면) 상관관계 차트
- "HRV가 낮은 날 에너지가 낮다고 느끼는 경향이 있어요"
- Oura의 "Tags" + Whoop의 "Journal" 패턴

## 4. 추가 제안

### 4-A. Progressive Disclosure 전면 적용

**핵심 원칙**: 모든 카드를 "한 줄 요약 → 탭하면 확장 → 네비게이션으로 상세"의 3단계로

현재 문제: 카드마다 정보 밀도가 달라서 스크롤 리듬이 불규칙
해법: 모든 카드가 collapsed 상태에서 동일한 높이 → 관심 있는 것만 펼침

```
Collapsed:  [🏋️ 트레이닝 준비도 82  ·  좋음  ›]
Expanded:   [🏋️ 트레이닝 준비도 82
             HRV ▲12%  수면 7.5h  피로 낮음
             추천: 중-고강도 운동 가능
             상세 보기 ›                      ]
```

### 4-B. Time-Aware Dashboard

대시보드가 시간대에 따라 자동 재구성:

| 시간대 | 강조 섹션 | 약화 섹션 |
|--------|----------|----------|
| 아침 (6-10) | 컨디션 + 모닝 브리핑 + 수면 리캡 | 운동 기록 |
| 낮 (10-17) | 활동 진행 + 운동 추천 | 수면 |
| 저녁 (17-22) | 하루 요약 + 내일 프리뷰 | 모닝 브리핑 |
| 밤 (22-6) | 수면 가이드 + 취침 알림 | 운동 추천 |

구현: 섹션 순서와 가시성을 시간대별로 동적 조정. 콘텐츠 자체는 동일하되 우선순위만 변경.

### 4-C. Widget-Like Mini Cards

Apple Widget 디자인 언어를 대시보드 카드에 차용:

- 각 카드가 독립적인 "위젯"처럼 동작
- 드래그로 순서 변경 가능
- 길게 눌러 크기 변경 (Small / Medium / Large)
- Widget Gallery처럼 "카드 추가/제거" 에디터

### 4-D. Cumulative Stress Score (장기 스트레스 지표)

Oura의 가장 혁신적인 2025 기능을 벤치마크:

- HRV 변동 + 수면 연속성 + 체온 변화 + 활동량을 30일 윈도우로 종합
- "스트레스가 축적되고 있어요" vs "잘 관리되고 있어요"
- 단기 컨디션과 다른 장기 추세 제공
- Wellness 탭의 새로운 히어로급 지표 후보

### 4-E. AI Daily Digest (자동 일일 리포트)

하루가 끝날 때 자동 생성되는 1-paragraph 요약:

```
"오늘은 컨디션 85로 시작해서 오전에 상체 운동 45분을
완료했어요. HRV가 운동 후 15% 떨어졌지만 오후에
빠르게 회복되었습니다. 수면 부채가 1시간 남아있으니
오늘은 11시 전에 취침을 추천합니다. 이번 주 운동
목표까지 2회 남았어요."
```

- FoundationModelReportFormatter 활용 (이미 존재)
- 푸시 알림으로 저녁에 전달
- NotificationInbox에 아카이브

### 4-F. Cross-Tab Deep Link

각 카드에서 관련 탭의 상세 화면으로 직접 이동:

| Today 카드 | → 이동 대상 |
|-----------|-----------|
| 수면 부채 | Sleep 탭 상세 |
| 운동 추천 | Activity 탭 추천 |
| 체중 카드 | Wellness 탭 체성분 |
| 습관 링 | Life 탭 |

현재 NavigationDestination이 MetricDetailView로만 가는 경우가 많은데, 각 탭의 고유 상세 뷰로 직접 연결하면 탭 전환 없이 깊이 탐색 가능.
