# Brainstorm: 평균 취침 시간 30분 전 Apple Watch 미착용 알림

## Problem Statement

사용자가 평소 취침 전 루틴에서 Apple Watch를 착용하지 않으면 수면 추적 데이터가 누락될 수 있다.
특히 취침 직전에는 사용자가 바쁜 경우가 많아 착용 여부를 놓치기 쉬워, "평균 취침 시간 30분 전"에 미착용 상태를 감지해 알려주는 기능이 필요하다.

## Target Users

- Apple Watch로 수면 추적을 하는 사용자
- 취침 시간이 비교적 일정하지만, 가끔 착용을 깜빡하는 사용자
- 다음 날 수면 데이터 누락을 최소화하고 싶은 사용자

## Success Criteria

1. 사용자의 평균 취침 시간 기반으로 알림 시점을 개인화할 수 있다.
2. 평균 취침 시간 30분 전 시점에 "미착용"일 때만 알림이 발송된다.
3. 이미 착용 중이면 불필요한 알림이 오지 않는다.
4. 알림을 탭하면 사용자가 즉시 착용할 수 있도록 간단한 안내를 제공한다.
5. 과도한 오탐/중복 알림 없이 사용자 불편을 최소화한다.

## Proposed Approach

- 최근 N일(예: 7일/14일)의 실제 취침 시작 시각을 기반으로 평균 취침 시간 계산
- 매일 1회, `평균 취침 시간 - 30분`에 로컬 알림 스케줄
- 알림 직전/알림 시점에 Watch 착용 상태를 확인해 조건부 발송
- 미착용이면 "수면 추적을 위해 지금 Apple Watch를 착용하세요" 안내
- 착용 중이면 해당 알림 skip

## Constraints

- 수면 시작 시각 추정의 정확도(사용자마다 변동 폭 존재)
- Apple Watch 착용 상태를 iPhone 앱에서 실시간/정확하게 확인 가능한지 기술 검증 필요
- HealthKit/알림 권한 거부 시 기능 동작 제한
- 배터리/백그라운드 실행 제약에 따른 상태 확인 타이밍 한계

## Edge Cases

- 평균 취침 시간이 아직 안정적으로 계산되지 않는 신규 사용자
- 주중/주말 취침 시간이 크게 다른 사용자
- 야간 근무 등으로 취침 시간이 수시로 바뀌는 사용자
- iPhone과 Watch 연결 해제/배터리 부족/비행기 모드
- 사용자가 이미 침대에 누운 후 알림을 늦게 보는 경우
- 수면 추적이 필요 없는 날(낮잠 위주, 여행 중 등)

## Scope

### MVP (Must-have)

- [ ] 최근 취침 데이터 기반 평균 취침 시간 계산
- [ ] 평균 취침 30분 전 1회 알림 스케줄
- [ ] 알림 시점 미착용 조건 체크
- [ ] 미착용 시에만 알림 노출
- [ ] 기능 On/Off 토글 + 기본 설명 문구

### Nice-to-have (Future)

- [ ] 평일/주말 분리 평균 취침 시간
- [ ] 사용자 지정 알림 리드타임(15/30/45분)
- [ ] 연속 미착용 시 부드러운 리마인드 전략(빈도 제한 포함)
- [ ] "오늘은 알림 끄기" 같은 당일 스누즈 옵션
- [ ] Watch 측 컴플리케이션/스마트 스택 연동

## Open Questions

1. 평균 취침 시간 계산 기준 기간은 며칠이 적절한가요? (7/14/30일)
2. 취침 시간 데이터 소스는 HealthKit 수면 카테고리만 사용할까요?
3. 미착용 판별은 어떤 신호를 신뢰할까요? (iPhone-Watch 연결 상태, 최근 손목 감지 이벤트 등)
4. 알림은 하루 1회 고정이 좋을까요, 조건에 따라 재알림이 필요할까요?
5. 사용자 제어 범위는 어디까지 열어둘까요? (리드타임 커스터마이즈, 요일별 설정)
6. 알림 탭 시 이동할 화면은 어디가 가장 자연스러울까요? (수면 설정, Watch 연동 가이드 등)
