# Brainstorm: 유산소 무움직임 감지 시 종료 추천 UX + 햅틱 피드백

## Problem Statement

Watch 유산소 세션에서 사용자가 실제로 운동을 끝냈어도 세션이 계속 기록되는 경우가 있다. 반대로 신호 대기/정수기 휴식처럼 **잠깐 멈춘 상황**도 빈번해, 무조건 자동 종료하면 오탐으로 사용자 신뢰를 해칠 수 있다.

핵심 과제는 다음 두 가지를 동시에 만족하는 것이다.

1. 운동 종료를 빠르게 정리할 수 있도록 가이드한다.
2. 잠깐 멈춤을 종료로 오인하지 않는다.

## Target Users

- 실외 러닝/걷기/사이클 사용자
- 인터벌 등 정지-재시작이 잦은 유산소 사용자
- Watch에서 운동 종료를 깜빡하는 사용자

## Success Criteria

1. 무움직임 이후 사용자가 1~2번 탭으로 쉽게 종료 또는 지속을 선택할 수 있다.
2. 오탐(잠깐 정지인데 종료 추천) 비율이 낮다.
3. 사용자 무응답 시에도 세션이 무한정 열려 있지 않다.
4. 햅틱만으로도 현재 상태(알림/확인/종료)가 구분된다.

## Proposed UX Flow (3-step)

### 1) Soft Nudge

- 조건(예시): 75초 무움직임
- 메시지: "움직임이 감지되지 않아요. 계속 운동 중인가요?"
- CTA: `계속` / `일시정지` / `종료`
- 햅틱: 가벼운 단일 알림

### 2) Confirmation

- 조건(예시): Soft Nudge 이후 추가 45초 무움직임
- 메시지: "계속 움직임이 없어 10초 후 종료할게요."
- UI: 10초 카운트다운
- CTA: `계속` / `지금 종료`
- 햅틱: 중간 강도 2회(리듬 패턴)

### 3) Auto-end with Undo Window

- 조건: 카운트다운 종료 + 무응답
- 동작: 세션 종료 및 저장 플로우 진입
- 메시지: "운동을 종료했어요. 필요하면 바로 재개할 수 있어요."
- CTA: `재개`(예: 30초 유예) / `확인`
- 햅틱: 종료 확정 패턴 1회

## Motion Detection Signal Strategy

무움직임 판단은 단일 신호가 아닌 **복합 신호**로 결정한다.

- 가속도/자이로: 손목 움직임
- 위치 변화(GPS): 실외 이동 유무
- 심박 추세: 운동 중 생리적 반응 유지 여부
- 최근 사용자 인터랙션: 화면 탭/디지털 크라운

### Guardrails

- 심박이 여전히 고강도 구간이면 종료 추천 강도를 낮춤
- GPS 이동이 계속되면 무움직임 타이머를 연장
- 일시정지 상태에서는 종료 추천 타이머 중단

## Haptic Feedback Mapping

| 상태 | 목적 | 패턴 |
|---|---|---|
| Soft Nudge | 알림 | Light 1회 |
| Confirmation | 주의 환기 | Medium 2회 |
| Auto-end 완료 | 상태 확정 | Firm 1회 |

원칙:
- 동일 의미 = 동일 패턴(일관성)
- 단계 상승에 따라 강도/주목도 증가
- 운동 중 과도한 방해를 피하기 위해 짧고 명확하게 유지

## Copy Draft (ko)

- Soft Nudge: "움직임이 잠시 멈췄어요. 운동을 계속하시나요?"
- Confirmation: "계속 움직임이 감지되지 않아 곧 종료할게요."
- Auto-end: "운동을 종료했어요. 잘못 종료됐다면 지금 재개할 수 있어요."

## Settings / Personalization

- "무움직임 종료 추천" ON/OFF
- 기준 시간 프리셋(보수적/표준/민감)
- 진동 강도 선택(약/중/강)
- 실내/실외 활동별 기본값 분리

## Edge Cases

- 인터벌 휴식 구간: 훈련 모드 인지 시 기준 시간 자동 연장
- 신호 대기(도심 러닝): GPS 정지 + 심박 유지 조합 고려
- 배터리 세이브/저전력: 고비용 센서 사용 축소 시 판단 보수적으로 변경
- Always-On/저조도: 복잡 UI 대신 단순 CTA + 햅틱 유지

## Scope

### MVP

- 3단계 종료 추천 플로우
- 기본 햅틱 3종 매핑
- 무응답 자동 종료 + 짧은 재개 유예

### Future

- 개인별 패턴 학습(종료 추천 타이밍 적응)
- 활동 유형별 별도 모델(러닝/사이클/로잉)
- 오탐/미탐 telemetry 기반 threshold auto-tuning
