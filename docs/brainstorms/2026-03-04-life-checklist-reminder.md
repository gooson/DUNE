# Brainstorm: 라이프 기록 개선 - 주기 체크리스트 알림

## Problem Statement
일상 관리 항목(예: 정수기 교체, 분리수거, 헤어샵 방문)을 주기적으로 실행해야 하지만, 기억 의존으로 누락되기 쉽다.  
주기 습관을 체크리스트로 관리하고, 완료 이후 다음 주기를 자동 계산해 적절한 시점에 반복 알림을 제공하는 기능이 필요하다.

## Target Users
- 본인 1인 사용자
- "생활 관리 루틴화"가 가장 중요한 목표
- 단순 입력과 빠른 완료 체크를 선호

## Success Criteria
- 사용자가 주기 항목을 등록하고, 완료 체크를 지속할 수 있다.
- 완료 시점 기준으로 다음 예정일이 자동 재계산된다.
- 다음 주기 전에 여러 번 알림이 전달되어 누락 빈도가 감소한다.
- 과거 완료 이력을 통해 루틴 유지 여부를 확인할 수 있다.

## Proposed Approach
1. 반복 규칙 기반 체크리스트 항목 생성
- 반복 단위: 주/월/분기(예: 1주, 1개월, 3개월)
- 항목 예시:
  - 정수기 교체: 3개월
  - 분리수거: 1주
  - 헤어샵: 1개월

2. 완료 체크 + 다음 주기 자동 계산
- 사용자가 항목을 완료하면 `완료일 + 반복 규칙`으로 다음 예정일 계산
- 고정 캘린더 기준이 아닌 완료일 기준 재계산 정책 적용

3. 다중 알림 지원
- 다음 예정일 전/당일에 여러 번 알림 스케줄링
- 예: 3일 전 + 1일 전 + 당일 오전

4. 이력 및 상태 관리
- 완료 히스토리(완료일, 메모, 지연 여부) 저장
- 미루기/건너뛰기 액션 제공
  - 미루기: 다음 예정일을 지정 기간만큼 이동
  - 건너뛰기: 해당 회차 미완료 처리 후 다음 회차로 진행

## Constraints
- 1인 사용자 중심이므로 협업/공유 기능은 초기 범위에서 제외
- 알림 피로도를 줄이기 위해 다중 알림의 기본값/상한 설계 필요
- 완료일 기준 재계산으로 인해 사용자별 주기가 유동적으로 변할 수 있음

## Edge Cases
- 예정일보다 늦게 완료한 경우: 완료일 기준으로 다음 주기 재설정
- 장기간 미완료 후 한 번에 완료한 경우: 백로그 회차를 자동 생성할지 여부 정의 필요
- 미루기 반복 사용 시: 과도한 지연을 허용할지 정책 필요
- 건너뛰기 후 연속 건너뛰기: 히스토리 표기와 알림 재개 시점 명확화 필요
- 알림 수신 실패(권한 해제, 기기 상태) 시: 앱 내 배지/인앱 리마인드 보강 필요

## Scope
### MVP (Must-have)
- 완료 체크
- 다음 알림
- 히스토리
- 미루기/건너뛰기
- 반복 규칙

### Nice-to-have (Future)
- 습관별 통계(연속 달성, 월별 완료율)
- 컨텍스트 알림(위치/시간대 기반)
- 템플릿 추천(생활 카테고리별 기본 루틴)

## Open Questions
- 다중 알림 기본 정책은 무엇으로 시작할지? (예: 3일 전/1일 전/당일)
- 미루기 최대 횟수 또는 최대 기간 제한이 필요한지?
- 건너뛰기 시 히스토리 상태를 "skip"으로 별도 표시할지?
- 히스토리 보관 기간을 무제한으로 둘지, 최근 N개월로 제한할지?
