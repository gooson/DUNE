# Brainstorm: 유산소 폰/워치 기록 및 상세 UX 정합성 개선

## Problem Statement

2026년 3월 2일 기준, 앱 내부에서 시작한 유산소 기록의 상세 화면이 운동 타입/기기(iPhone vs Apple Watch)에 따라 다르게 노출된다.

- `달리기`(iPhone 기록): 시간/칼로리만 보이고 km, 심박, 페이스 등 핵심 지표가 누락
- `걷기`(iPhone 기록): km/심박/페이스가 정상 노출
- 사용자 기대: Apple Fitness(워치 기준)와 동일한 구조를 기본으로, iPhone은 가능한 경우 더 많은 지표 제공

핵심 문제는 **데이터 수집 경로와 UI 표시 규칙이 운동 타입별로 일관되지 않아, 동일한 유산소 경험이 깨지는 것**이다.

## Target Users

- Apple Watch + iPhone 병행 사용자
- 운동 기록의 신뢰도와 일관성을 중요하게 보는 사용자
- 기기/운동 타입이 달라도 같은 기준으로 비교하고 싶은 사용자

## Success Criteria

1. 유산소 상세 화면이 Apple Fitness(워치 기준) 정보 구조와 최대한 동일하게 동작한다.
2. iPhone 기록에서도 사용 가능한 지표는 최대한 노출한다("다다익선").
3. 데이터가 없을 때는 숨김 대신 **명확한 안내 문구**를 표시한다.
4. 앱 내부 시작 기록(폰)과 워치 동기화 기록 모두에서 지표 계산/표시 기준이 일관된다.
5. 달리기/걷기 등 주요 유산소 타입 간 표시 편차가 제거된다.

## Proposed Approach

### 1. 데이터 모델 정합성 우선 정리

- Apple Fitness 기준으로 유산소 핵심 지표 세트를 정의한다.
- Workout 샘플 + 관련 quantity sample(거리, 심박, 에너지, 페이스 계산 원천) 매핑을 타입별로 명시한다.
- 앱 내부 시작(iPhone) 경로에서 달리기/걷기의 저장 필드가 동일 규칙을 따르는지 검증한다.

### 2. 상세 화면 표시 규칙 통일

- 표시 우선순위를 "지표 존재 여부" 기반으로 통일한다(운동 타입 하드코딩 최소화).
- `달리기`와 `걷기`가 같은 카테고리 규칙을 타도록 공통 카드 레이아웃/뷰모델을 정리한다.
- iPhone은 센서/권한 범위 내에서 노출 가능한 추가 지표를 확장한다.

### 3. 데이터 없음 UX 표준화

- 누락 지표는 무조건 빈 공간이 아닌 안내 문구로 대체한다.
- 안내 메시지는 원인 중심으로 통일한다.
  - 예: 심박 미연결/권한 미허용/해당 기록에 샘플 없음
- 안내는 정보 카드 내 인라인 방식으로 노출한다(경고성 과잉 표현 지양).

### 4. 검증 기준을 Apple Fitness 참조로 고정

- 동일 기록에 대해 앱 화면과 Apple Fitness 표시 항목을 비교 검증한다.
- iPhone 단독 기록/워치 기록/혼합 시나리오를 분리해 회귀 테스트한다.

## Constraints

- 기준 제품: Apple Fitness 앱 동작 방식 준수
- 데이터 소스: HealthKit 중심(추정값 남용 금지)
- 범위: UX + 기록(수집/매핑/표시) 모두 포함
- 병행 사용자 경험을 해치지 않도록 iPhone/Watch 간 표시 규칙을 통합해야 함

## Edge Cases

1. 초단시간 운동(0~1분)으로 거리/심박 샘플이 늦게 들어오는 경우
2. iPhone 단독 운동 중 심박 센서 데이터가 없는 경우
3. 권한 일부 거부(심박만 미허용 등)로 지표가 부분 누락되는 경우
4. 실내/실외에 따라 거리 샘플 품질이 크게 차이나는 경우
5. 앱 내부 시작 직후 종료로 summary 계산이 비정상(0분/소수 칼로리)인 경우
6. 운동 타입별(달리기/걷기/사이클) 지표 지원 범위가 다른 경우

2. "데이터 없음" 안내 문구의 로컬라이즈/톤 정책(간결형 vs 원인 상세형) 확정 필요
