# Brainstorm: 알림시스템 다각화

## Problem Statement

현재 알림이 일부 이벤트에 편중되면 사용자마다 중요도가 다르게 느껴져 알림 피로가 생길 수 있다.
알림 채널, 트리거, 개인화 강도를 다각화해 "적게 보내도 더 유의미한" 알림 경험으로 전환해야 한다.

## Target Users

- 1차 타겟: 기록/PR 중심 파워 유저
- 고빈도 운동 사용자 (주 4회 이상)
- 수치 기반 피드백(HRV, RHR, 수면, PR)을 선호하는 사용자
- Watch와 iPhone을 함께 사용하는 멀티 디바이스 사용자

## Success Criteria

1. 7일/30일 리텐션 개선 (최우선 KPI)
2. 파워 유저 세그먼트의 주간 활성일 수 유지 또는 증가
3. 알림 OFF 전환율 및 즉시 삭제율 감소
4. 동일 사용자 기준 주간 알림량은 유지 또는 감소
5. 푸시 알림 클릭률(CTR) 상승

## Proposed Approach

### 1) 채널 다각화

- 잠금화면 푸시: 즉시 행동이 필요한 이벤트(PR, 이상 징후)
- 위젯/배지: 비침습 상태 알림
- Watch 미러링: 운동/건강 핵심 이벤트만 전달

`인앱 알림 허브`: Future 범위로 이관 (MVP 제외)

### 2) 트리거 다각화

- 성과형: PR, 연속 기록(streak), 주간 목표 달성
- 건강형: baseline 대비 이탈(HRV, RHR, 수면 부족)
- 루틴형: 사용자의 실제 운동 시간대 기반 리마인더
- 회복형: 고강도 운동 다음날 회복 권장
- 컨텍스트형: 날씨/시간대 기반 실외 운동 대체 제안

### 3) 개인화/피로 제어

- 사용자별 `알림 예산` 도입 (일/주 최대 전송량)
- 유형별 빈도 캡 (건강형 1일 1회, PR은 즉시)
- 최근 반응률 기반 자동 감쇠 (무반응 유형은 빈도 감소)
- Quiet Hours, 집중 시간대, 주말/평일 선호 반영

### 4) 메시지/행동 설계

- 단순 정보 대신 행동 제안 포함: "오늘은 20분 저강도 걷기 추천"
- 알림 액션 버튼: 시작, 30분 후 다시 알림, 오늘 건너뛰기
- 이유 설명형 카피: "지난 7일 평균 대비 HRV -22%"

### 5) 운영/실험 프레임

- 템플릿 기반 알림 카테고리 운영 (성과/건강/루틴/회복)
- A/B 테스트: 전송 시간, 카피 길이, 액션 버튼 유무
- 전역 kill switch 및 카테고리별 긴급 비활성화

## Constraints

- MVP는 로컬 알림 기반으로 제한 (서버 연동 없음)
- 서버 기반 세분화/실험은 Future 범위로 이관
- HealthKit 데이터 지연/누락 가능성
- 다국어 메시지 및 수치 포맷 일관성 필요
- iPhone/iPad/Watch 라우팅 정책 통합 필요

## Decisions

- 1순위 KPI: 리텐션
- 1차 타겟: 파워 유저
- MVP 아키텍처: 로컬 알림 only
- 우선 리스크: 알림 피로 + 중복 알림
- 범위 결정: 푸시 최적화만 먼저 진행 (인앱 허브 제외)

## Edge Cases

- 우선순위 1: 알림 피로 (과도한 빈도, 낮은 신호대잡음)
- 우선순위 2: 중복 알림 (동일 이벤트 반복 전송)
- 권한 거부 또는 중간 철회 시 대체 UX 필요
- 데이터 부족 사용자(신규 설치)에서 baseline 계산 불가
- 중복 이벤트 발생 시 동일 알림 반복 발송 위험
- 해외 이동/시간대 변경 시 알림 시간 왜곡
- 운동 기록 삭제 후 딥링크 타겟 미존재

## Scope

### MVP (Must-have)

- [ ] 알림 카테고리 4종 (성과/건강/루틴/회복) 정의
- [ ] 카테고리별 빈도 캡 + 사용자별 일일 알림 예산 (피로 제어 핵심)
- [ ] 최소 2개 개인화 규칙 (사용 시간대, 반응률 감쇠)
- [ ] 푸시 dedup 키 도입 + 중복 방지 윈도우(예: 60분)
- [ ] 우선순위 기반 전송 정책 (PR/이상 징후 우선, 정보성 후순위)
- [ ] 알림 설정 화면에서 카테고리 on/off 및 Quiet Hours

### Nice-to-have (Future)

- 서버 기반 실시간 세그먼트/실험 플랫폼
- 인앱 알림 허브 + 읽음/미읽음 + 딥링크 라우팅
- 주간 요약 리포트(알림 성과, 건강 인사이트)
- 코치형 시나리오(주간 목표 재설정 제안)
- 소셜/챌린지 기반 알림

## Open Questions

1. 알림량 상한(일/주)은 사용자별 고정값으로 시작할지 적응형으로 시작할지?
2. 강한 개입(이상 징후)과 약한 개입(정보성)의 비율을 어떻게 가져갈지?
