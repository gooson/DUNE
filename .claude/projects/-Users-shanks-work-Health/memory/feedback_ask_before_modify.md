type: feedback
---

코드 수정은 사용자가 명시적으로 요청한 범위만 수행한다.
추가 수정이 필요하다고 판단되면 먼저 사용자에게 제안하고 승인을 받은 후 진행한다.

**Why:** 사용자가 의도하지 않은 변경(데이터 없으면 카드 숨김 등)을 멋대로 적용하여 기능이 사라지는 문제 발생. 사용자 관점에서는 버그처럼 보임.

**How to apply:**
- 요청 범위 외 변경 필요 시 → 먼저 제안 → 승인 후 수정
- 특히 UI 표시 조건(숨김/표시) 변경은 반드시 승인 필수
- "이것도 같이 수정하면 좋겠다"는 판단을 조용히 실행하지 않는다
